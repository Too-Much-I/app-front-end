# 모의고사 파트 가이드 노출 상태

## 현재 동작

로그인과 실제 시험 진행 화면이 아직 연결되지 않았으므로, 모의고사 준비 화면 헤더의 도움말 버튼으로 파트 가이드에 수동 진입한다. 현 단계에서는 열람 여부를 저장하거나 가이드를 자동으로 열지 않는다.

## 결정

로그인 도입 후 "회원가입 뒤 첫 모의고사 화면 진입"을 계정 기준으로 판단하려면 서버 상태를 진실의 원천으로 사용한다. 기기 로컬 저장소만 사용하면 재설치, 앱 데이터 삭제, 다른 기기 로그인 때 열람 상태가 사라지므로 최초 1회를 보장할 수 없다.

단순 boolean 대신 사용자가 확인한 가이드 버전을 저장한다.

```ts
interface UserGuideState {
  mockExamGuideVersionSeen: number;
}

const CURRENT_MOCK_EXAM_GUIDE_VERSION = 1;
```

서버 값이 현재 앱의 가이드 버전보다 낮으면 가이드를 자동으로 연다. 버전 방식은 시험 형식이나 필수 안내가 크게 바뀌었을 때 기존 사용자에게 새 가이드를 한 번 다시 보여줄 수 있다는 장점이 있다.

## 권장 흐름

1. 로그인 완료 시 사용자 프로필과 `mockExamGuideVersionSeen`을 받는다.
2. 사용자가 모의고사 준비 화면에 진입한다.
3. 서버 버전이 현재 가이드 버전보다 낮으면 파트 가이드를 자동으로 연다.
4. 사용자가 마지막 페이지에서 `시작하기`를 누르면 서버에 현재 버전을 저장한다.
5. 저장 결과와 함께 실제 모의고사 세션 생성 및 응시 화면 이동을 진행한다.

첫 페이지가 표시되자마자 열람 완료로 처리하지 않는다. 진입 직후 앱을 종료한 사용자가 이후 안내를 다시 볼 수 있어야 하므로, 마지막 페이지의 명시적인 행동을 완료 시점으로 삼는다.

## 로컬 상태의 역할

로컬 저장소는 서버 상태를 대체하지 않고 화면 깜빡임 방지나 오프라인 재시도용 캐시로만 사용할 수 있다. 사용자별 키로 분리하고, 서버 저장에 실패한 상태는 다음 온라인 시점에 재시도해야 한다.

시험 응시 이력 유무로 가이드 열람 여부를 추론하지 않는다. 가이드 열람과 시험 응시는 서로 다른 사건이며, 가이드를 본 뒤 시험을 시작하지 않은 사용자를 올바르게 구분할 수 없기 때문이다.

## 도입 시 확인할 사항

- 신규 회원의 서버 기본 버전은 `0`으로 둔다.
- 기존 회원에게도 가이드를 다시 보여줄지 정한 뒤 마이그레이션 기본값을 결정한다.
- 인증 프로필 응답에 버전을 포함할지 별도 온보딩 상태 API를 둘지 백엔드와 합의한다.
- 서버 조회가 실패하더라도 수동 도움말 진입은 항상 사용할 수 있게 유지한다.
- 실제 시험 화면은 준비·가이드 화면과 달리 하단 탭바를 숨긴다.
